혹시 이런 경험, 있으신가요? 밤새워 20장의 슬라이드를 만들었는데, 다음 날
아침에 다시 보니 스토리의 흐름이 영 마음에 들지 않는 경험 말입니다.
회의시간에 상사의 지적은 항상 똑같습니다. "그래서 결론이 뭐야? 핵심이
없어." 결국 공들여 만든 슬라이드 대부분을 버리고 처음부터 다시 만드는
악순환이 반복됩니다. 왜 이런 상황이 계속될까요? 설계도 없이 곧바로
인테리어부터 시작했기 때문입니다.

데이터 시각화는 집을 짓는 과정과 같습니다. 프로 건축가는 공사를 시작하기
전에 설계도를 수십 번 고칩니다. 종이 위에서 고치는 것이 이미 지어진
건물을 허물고 다시 짓는 것보다 훨씬 빠르고 효율적이기 때문입니다.
마찬가지로, 프로 일잘러는 파워포인트를 열기 전에 먼저 스토리의 구조를
설계합니다. 슬라이드를 다 만든 뒤 순서를 바꾸고 내용을 고치는 것보다,
백지 위에서 생각의 구조를 다듬는 편이 훨씬 효율적이니까요.

'생각의 지도'를 그리는 것은 절대 번거로운 추가작업이 아닙니다. 오히려
우리의 야근을 막아주고, 단번에 OK를 받는 보고서를 만드는 핵심과정입니다.
복잡한 생각을 명확하게 정리해 주는 생각정리의 시간이죠. 이제 우리는 AI의
힘을 빌려, 가장 빠르고 똑똑한 방법으로 이 '생각의 지도'를 그리는 3단계
프로세스를 알아보겠습니다.

\

#### ['생각의 지도'를 만드는]{.u2} {#013.xhtml_sigil_toc_id_7 title="‘생각의 지도’를 만드는 3단계 프로세스"}

[3단계 프로세스]{.u2}

\

**1단계. 제미나이에게 '심층분석'을 맡겨 재료 확보하기**

첫 단계는 AI에게 '탐험가'의 역할을 맡기는 겁니다. 주제에 대한 배경, 시장
동향, 경쟁사 사례, 실행 아이디어, 그리고 예상 반론까지 가능한 자료를
최대한 폭넓게 수집하게 합니다. 마치 컨설턴트에게 '이 주제에 대해 리서치
보고서를 써달라'고 요청하는 것과 같습니다. 제미나이에게 이렇게 요청해
보세요.

\

::: tb1
[프롬프트 예시]{.cor1}

\

너는 최고의 마케팅 전략 컨설턴트야. '유튜브 숏폼을 활용한 Z세대 마케팅
전략'에 대해 심층분석[(Deep Research)]{.m}을 해줘. 시장 동향, 경쟁사
사례, 실행 아이디어, 예상 반론, 핵심성과지표[(KPI)]{.m}를 포함해
다각도로 분석해 줬으면 해.
:::

\

이렇게 지시하면 제미나이는 머릿속으로 모든 리서치를 끝내고, 깊이 있는
분석 결과를 보고서 형태로 보여줍니다[(이때 입력창 하단의 '도구' 메뉴에서
'Deep Research'를 활성화해야 합니다)]{.m}.

\

::: w100
![](AI로 차트 그리기/Images/35.jpg){.w100}
:::

\

그런데 여기서 "챗GPT나 퍼플렉시티로도 되지 않나요?"라는 질문이 나올 수
있습니다. 물론 가능합니다. 다만 이번 단계의 목표는 단순히 정보를 찾는
것을 넘어, 정보를 바탕으로 아이디어를 구상하고 '마인드맵으로 구조화'하는
복합적인 작업입니다. 퍼플렉시티가 '정보 검색'에 강하다면, 제미나이는
'리서치+기획+정리'를 한 번에 수행하는 최고의 올라운더입니다. 특히
제미나이는 구글 검색과 연결되어 있어, 최신 시장동향이나 최근 경쟁사
사례처럼 실시간 정보가 중요한 리서치에서 뛰어난 정확도를 보여줍니다.

\

::: w100
![](AI로 차트 그리기/Images/36.jpg){.w100}
:::

\

**2단계. 제미나이에게 '마인드맵 요약'을 지시하여 뼈대 세우기**

AI가 가져온 리서치 결과를 그대로 사용하지 말고, 이번에는 구조화된 형태로
다시 정리하도록 지시합니다. AI에게 한 번에 모든 일을 시키는 것보다
이렇게 단계를 나누어 요청하는 것이 훨씬 안정적이고, 결과물의 품질도
높습니다.

\

::: tb1
[프롬프트 예시]{.cor1}

\

좋아. 방금 분석한 내용을 바탕으로 '마인드맵'을 그리려고 해.\
'Z세대 숏폼 전략'을 중심 주제로 잡고, 아래 가지들에 맞춰 핵심 키워드로
요약해 줘.

• 주요 가지 : 시장 분석, 콘텐츠 전략, 채널 운영, 성과 측정, 리스크 관리

**\[중요 : 출력규칙\]**

1\. Markmap 시각화 도구에 바로 넣을 수 있도록 '마크다운 코드 블록' 안에
작성해 줘.

2\. "네, 알겠습니다" 같은 잡담이나 불필요한 설명은 생략하고, 오직 코드만
출력해 줘.

3\. 계층 구조는 \# [(제목)]{.m}과 - [(내용)]{.m} 기호만 사용해 줘.
:::

\

여기서 중요한 포인트는 '마크다운[Markdown]{.m1}'이라는 형식입니다.
마크다운은 AI가 가장 이해하기 쉬운 '구조화된 글쓰기 언어'로,
들여쓰기만으로 계층을 표현할 수 있습니다. 마크다운을 사용하는 이유는
다른 시각화 도구들과 소통할 수 있는 '공용 설계도'이기 때문입니다. 즉,
제미나이가 마크다운을 사용해 마인드맵 형태로 내용을 정리하면 다음
단계에서 사용해 볼 마크맵[Markmap]{.m1} 같은 시각화 도구들이 그 구조를
그대로 읽어들일 수 있습니다.

\

::: w100
![](AI로 차트 그리기/Images/37.jpg){.w100}
:::

**3단계. 마크맵으로 텍스트 설계도를 '마인드맵'으로 변신시키기**

이제 2단계에서 얻은 마크다운 텍스트를 시각적인 형태로 바꿔 보겠습니다.
아무리 훌륭한 텍스트라도 시각화가 없으면 설득력이 약해집니다. 이제
마지막 마법으로, 텍스트를 마크맵[Markmap]{.m1}에 붙여넣어 그래픽
마인드맵으로 10초 안에 변신시켜 보겠습니다.

\

**(1) 마크맵 웹사이트 접속하기**

웹 브라우저에서 마크맵[markmap.js.org]{.m1}에 접속합니다. 그리고 화면
중앙에 보이는 'Try it out **→**' 링크를 클릭하면 마크다운을 편집할 수
있는 화면으로 이동합니다. 왼쪽에는 텍스트 입력창, 오른쪽에는 마인드맵
미리보기가 나타납니다.

\

::: w100
![](AI로 차트 그리기/Images/38.jpg){.w100}
:::

\

**(2) 마인드맵 텍스트 복사하기**

왼쪽 텍스트 입력창에는 기본으로 작성된 예시 코드가 있는데, 이 부분을
모두 삭제합니다. 그리고 제미나이로 돌아와 답변창 아래에 있는 '대답
복사'를 클릭해 마크다운 형식의 마인드맵 텍스트 전체를
복사합니다[(마우스로 텍스트 전체를 긁어서(드래그) 복사하면 불필요한
빈칸이나 서식이 묻어서 마인드맵이 안 나올 수 있습니다)]{.m}.

\

::: w100
![](AI로 차트 그리기/Images/39.jpg){.w100}
:::

\

**(3) 마인드맵 만들기**

다시 마크맵 웹사이트로 돌아와 비어 있는 왼쪽 창에 제미나이에서 복사한
마크다운 텍스트를 그대로 붙여넣으면, 오른쪽에 자동으로 구조화된
마인드맵이 생성됩니다.

\

::: w100
![](AI로 차트 그리기/Images/38_1.jpg){.w100}
:::

\

어떤가요? 이제 여러분의 머릿속이 명확해졌나요? 어떤 순서로 발표를 해야
할지, 어떤 근거를 제시해야 할지, 모든 것이 한눈에 보이는 완벽한 '생각의
지도'가 완성된 겁니다.

\

#### [완성된 '생각의 지도' 활용하기]{.u2 title="완성된 ‘생각의 지도’ 활용하기"} {#013.xhtml_sigil_toc_id_8}

\

이렇게 만든 마인드맵은 단순히 보기 좋은 그림이 아닙니다. 우리의 발표
흐름을 정리하고, 청중의 이해를 돕는 강력한 설계도입니다.

\

**(1) 이미지로 저장하여 발표자료에 삽입하기(SVG 활용)**

마크맵 화면 오른쪽 아래에 있는 'SVG로 저장하기[Download as SVG]{.m1}'
버튼을 클릭해 이미지 파일로 다운로드합니다.

이렇게 다운로드한 SVG 파일을 파워포인트의 첫 페이지나 목차 페이지에
삽입하세요. 그리고 발표 시작 전에 "오늘 저는 이 지도에 따라
진행하겠습니다"라고 말하며 전체 그림을 먼저 보여주면, 발표의 설득력이 두
배로 높아집니다.

\

**(2) 링크로 공유하여 사전 브리핑하기(HTML 활용)**

마크맵 화면 오른쪽 아래에 있는 'HTML로 저장하기[Download as interactive
HTML]{.m1}' 버튼을 클릭하면 마인드맵 자체가 하나의 인터랙티브 웹페이지
파일로 저장됩니다.

중요한 회의 전에, 이 HTML 파일을 팀원들에게 미리 공유하세요. 팀원들은 각
항목을 직접 클릭하며 전체 구조를 미리 파악할 수 있으므로, 회의 중에는
불필요한 설명이 줄어들고 핵심논의에만 집중할 수 있습니다.

\

**(3) 발표의 내비게이션으로 사용하기**

발표할 때 듀얼 모니터 한쪽에 마인드맵을 항상 띄워두면 발표의 흐름을 잃지
않게 도와주는 내비게이션이 되어, 말이 빗나가거나 순서를 놓치는 실수를
방지해 줍니다.

\

자, 이제 머릿속에 있던 막연한 생각들이 AI 덕분에 하나의 완벽한 시각적
설계도로 정리되었습니다. 다음 장에서는 이 설계도를 기반으로, AI에게
'어떻게 요청해야 원하는 결과를 얻을 수 있는지', 즉 실전 프롬프트의
구조를 만드는 법을 알아보겠습니다.
