이 책을 읽고 있는 분들의 대부분은 코딩을 한 번도 배워본 적이 없을
겁니다. 괜찮습니다. 저도 그렇습니다. 그런데 우리는 파워포인트 같은
도구로 디자인을 하다 보면, 어느 순간 이런 갈증이 생기기 시작합니다.

'이미지에 마우스를 올렸을 때 살짝 움직이게 만들 수는 없을까?'

'버튼을 눌렀을 때 상세 설명이 아래로 쫙 펼쳐지면 얼마나 멋질까?'

'화면 전환이 물 흐르듯 자연스럽게 이어져서 사람들의 시선을 확 끌었으면
좋겠어.'

저 역시 이런 생각을 하며 여러 번 시도해 봤지만 거의 불가능에
가까웠습니다. 그때 깨달았습니다. '아, 이건 디자인 감각의 문제가 아니라,
움직임을 구현하는 기술의 문제구나.' 그 순간부터 제 머릿속에는
'코딩'이라는 단어가 맴돌았습니다. 이런 마법 같은 움직임을 구현하려면
자바 스크립트나 파이썬 같은 프로그래밍 언어를 공부해야겠다고
생각했습니다. 그래서 저도 야심 차게 코딩을 배우기에 도전했다가 검은 화면
속에 쏟아지는 낯선 명령어들 앞에서 주눅 들며 몇 번의 시도 끝에
포기했습니다.

하지만 이제는 다릅니다. AI가 우리의 '동시통역사'가 되어주는 시대입니다.
지금까지의 코딩이 '외국어를 직접 배워 외국인과 대화하는 것'이었다면, AI
시대의 코딩은 '유능한 통역사[(AI)]{.m}에게 한국어로 원하는 바를 명확하게
전달하는 것'과 같습니다. 이제는 더 이상 프로그래밍 언어를 배울 필요가
없습니다. 우리의 언어를 알아듣는 AI가 있기 때문이죠. AI에게 '이런
움직임을 구현해 줘'라고 일상 언어로 지시하면, AI가 '코딩'이라는 복잡한
언어로 뚝딱 번역해 주는 시대가 온 겁니다.

Part 3에서 우리는 '디렉터'가 되어 AI 통역사에게 지시를 내리고, 세상에
없던 나만의 맞춤 시각화를 만드는 짜릿한 경험을 하게 될 겁니다. 겁먹지
마세요. 여러분은 단 한 줄의 코드도 만들 필요가 없습니다. 그저 저를 믿고,
재미있는 놀이를 하듯 따라오기만 하면 됩니다. 코딩을 배우지 않아도,
코딩하듯 논리적으로 '생각'할 수 있다면 우리의 시각화 세계는 한층 더
넓고, 풍부하고, 재미있어질 겁니다.

우리는 매일 수많은 데이터와 정보의 홍수 속에서 일합니다. 어떤 일을 먼저
해야 할지, 어떤 데이터가 더 중요한지, 그리고 그것을 어떻게 팀원들에게
명확하게 공유해야 할지 막막할 때가 많죠. 이럴 때 필요한 것이 바로 '업무
시각화[Work Visualization]{.m1}'입니다. 업무 시각화는 생각을 눈에 보이는
구조로 바꾸어 복잡한 정보를 단순하게 정리하고, 팀의 방향을 하나로
맞춰주는 강력한 무기입니다.

'시각화는 그냥 PPT나 엑셀로 그리면 되는 거 아니야?'라고 생각할 수
있습니다. 물론 맞는 말입니다. 하지만 여기에 코딩을 활용한다면, 우리는
완전히 다른 차원의 업무 효율과 결과물을 경험할 수 있습니다. 마치 손으로
스케치를 하다가 전문적인 3D 설계도구[CAD]{.m1}를 사용하는 것처럼 말이죠.

이번 장에서는 코딩이 우리의 업무를 어떻게 자동화하고, 더 나아가 일하는
방식을 혁신할 수 있는지 쉽고 명확하게 안내하겠습니다.

#### [코딩, 대체 뭔가요?]{.u2} {#028.xhtml_sigil_toc_id_25}

\

흔히 '코딩'이라고 하면 검은 화면에 알 수 없는 영어가 가득 찬 모습을
떠올립니다. 하지만 코딩의 본질은 생각보다 단순합니다. 코딩은 '컴퓨터에게
지시하는 언어'이자, 동시에 '아주 구체적인 조립설명서'입니다. 예를 들어
우리가 "책상 하나 만들어 주세요"라고 말하면 사람은 알아서 재료를 찾아
조립을 시작합니다. 하지만 컴퓨터는 그렇지 못하죠. 그래서 컴퓨터에게는
다음과 같이 아주 구체적인 절차를 알려줘야 합니다.

\

\> \* A번 나사 4개와 B번 다리 4개를 준비한다.

\> \* C번 상판의 1, 2, 3, 4번 구멍에 B번 다리를 맞춘다.

\> \* 각 구멍에 A번 나사를 넣고 드라이버로 5바퀴씩 돌려 조인다.

\> \* \...

\

이처럼 어떤 결과물을 만들기 위해 필요한 절차와 규칙을 논리적으로
나열하고, 컴퓨터가 이해할 수 있는 언어[(Python, SQL, JavaScript
등)]{.m}로 명령을 내리는 과정이 바로 '코딩'입니다. 즉, 코딩은 단순한
기술이 아니라 논리적으로 사고하는 방법입니다. 복잡한 문제를 잘게 쪼개고,
순서대로 해결하는 '생각의 방식'이기도 하죠.

문제는 여기서부터입니다. '조립설명서[(논리)]{.m}라는 것은 알겠는데, 이걸
컴퓨터 언어로 쓸 줄 모르는데요?' 바로 이 지점에서 대부분의 일잘러들이
멈칫합니다. 이 거대한 진입장벽 때문에 우리는 코딩의 잠재력을 알면서도
엑셀과 PPT라는 익숙한 도구에 머물러 왔습니다.

'개발자도 아닌데, 내가 왜 코딩을 알아야 해?'라는 생각이 드는 건 너무나
자연스러운 반응입니다. 하지만 우리는 이미 수많은 디지털 도구와 함께
일하고 있습니다. 그래서 코딩을 안다는 것은, 우리가 매일 사용하는 도구의
원리를 이해하고, 나아가 그 도구를 내 입맛에 맞게 직접 만들거나 개선할 수
있는 힘을 갖게 됨을 의미합니다. 이는 마치 자동차 운전만 할 줄 아는 것과
자동차의 구조를 이해하고 간단한 정비까지 할 수 있는 것의 차이와
같습니다. 후자의 경우, 차에 문제가 생겼을 때 빠르고 정확하게 대처할 수
있으며, 차의 성능을 최대한으로 활용할 수 있겠죠. 마찬가지로 업무에
코딩을 적용하면 단순반복 업무는 컴퓨터에게 맡기고, 우리는 더 창의적이고
중요한 일에 집중할 수 있게 됩니다.

\

#### [AI라는 만능 비서 :]{.u2} {#028.xhtml_sigil_toc_id_26 title="AI라는 만능 비서 : 제미나이 vs 클로드"}

[제미나이 vs 클로드]{.u2}

\

이제 우리의 역할은 '직접 벽돌을 쌓는[(코딩하는)]{.m}' 것이 아니라,
'무엇을, 왜, 어떻게 지을지 설명하는[(프롬프팅하는)]{.m}' 디렉터로
바뀌었습니다. 이때 중요한 것은, 우리의 의도를 정확한
'설계도[(코드)]{.m}'로 번역해 줄 파트너를 고르는 일입니다. 그 역할을
가장 안정적으로 수행하는 AI가 바로 제미나이와 클로드입니다. 이들은
각자의 전문 분야가 다릅니다.

\

**• 제미나이**[Gemini]{.m1} : 최신 기술을 아는 '고속 프로토타이퍼',
제미나이는 구글 검색을 통해 세상의 모든 최신 건축자재[(최신 라이브러리,
API)]{.m}를 실시간으로 파악합니다. 간단한 스케치나
시제품[(프로토타입)]{.m}을 놀랍도록 빠른 속도로 만들어 내는 데 강합니다.

**• 클로드**[Claude]{.m1} : 복잡한 설계를 이해하는 '구조 기술자',
클로드는 한 번에 아주 두꺼운 설계도 묶음[(대용량 코드 파일, 여러
문서)]{.m}을 통째로 검토하는 데 탁월합니다. 이미 지어진 복잡한 건물(기존
코드)의 문제점을 찾고, 더 튼튼하게[(리팩토링)]{.m}, 더 안전하게[(보안
검토)]{.m} 개선하는 데 강점을 보입니다.

\

제미나이는 최신 도구를 누구보다 빨리 이해하고, 아이디어를 실물로
구현하는 속도가 놀랍습니다. 클로드는 이미 만들어진 코드를 정교하게
다듬고, 지속가능한 구조를 설계하는 데 강점을 보입니다. 결국 제미나이는
'속도', 클로드는 '안정성'입니다. 이 둘을 함께 활용하면, 아이디어 → 코드
→ 시각화까지의 흐름을 완벽히 연결할 수 있습니다.

\

#### [AI 코딩으로 어떤 결과물을]{.u2} {#028.xhtml_sigil_toc_id_27 title="AI 코딩으로 어떤 결과물을 만들 수 있을까요?"}

[만들 수 있을까요?]{.u2}

\

PPT나 엑셀로 만드는 시각화가 기성복이라면, 코딩으로 만드는 시각화는 내
몸에 꼭 맞는 맞춤 정장입니다. AI의 도움을 받으면, 이 맞춤 정장을 만드는
일도 결코 어렵지 않습니다. 코딩으로 시각화를 하면 3가지 장점이 있습니다.

\

**① 무한한 자유도** : 기존 툴에서 제공하는 정해진 차트 디자인이나
템플릿에 얽매일 필요가 없습니다. 내가 원하는 형태, 색상, 상호작용,
애니메이션 효과 등 모든 요소를 자유롭게 구현할 수 있습니다. 회사의
브랜딩 가이드에 딱 맞는 시각화 자료를 만들 수도 있죠.

**② 자동화와 실시간 업데이트** : 한 번 코드를 작성해 두면, 새로운
데이터가 들어올 때마다 자동으로 시각화 자료가 업데이트됩니다. 매번
차트를 새로 그리는 수고가 사라집니다.

**③ 인터랙티브 기능** : 정적인 이미지에 그치지 않습니다. 사용자가 직접
차트의 특정 부분을 클릭하거나 필터를 적용하며 데이터를 탐색할 수 있는
'살아 있는' 대시보드를 만들 수 있습니다. 이는 보고를 받는 사람이
데이터를 훨씬 더 깊이 있게 이해하도록 돕습니다.

\

코딩을 활용하면 생각보다 훨씬 다양하고 실용적인 시각화 결과물을 만들 수
있습니다. 대표적인 예시는 다음과 같습니다.

\

**① 인터랙티브 차트/그래프** : 마우스를 올리면 세부 정보가 표시되고,
클릭하면 특정 데이터 계열을 숨기거나 강조할 수 있는 동적인 차트를 만들
수 있습니다.

**② 웹 기반 대시보드** : 여러 차트와 지표를 한 화면에 모아 놓은 실시간
현황을 한눈에 파악할 수 있는 웹페이지를 구현합니다. 언제 어디서든
브라우저만 있으면 확인할 수 있습니다.

**③ 보고서 자동화시스템** : 정해진 시간마다 자동으로 데이터를
수집ㆍ분석ㆍ시각화하여 PDF나 HTML 파일 형태로 생성되는 보고서를 만들 수
있습니다.

\

코딩은 더 이상 '개발자만의 언어'가 아닙니다. 생각을 설계로, 설계를
시각화로 구체화하는 언어입니다. 그렇다고 우리가 코딩을 배울 필요는
없습니다. 우리에게 필요한 것은 '코딩적 사고'입니다. 이는 단순한 기술
습득이 아니라, 일을 설계하는 새로운 방식을 배우는 겁니다. 결국 코딩적
사고는 업무시간과 노력을 획기적으로 줄여주고, 더 정확하며, 더 깊이 있는
인사이트를 제공하는 지속가능한 시스템을 만드는 일입니다.

이제 다음 장에서 이 개념을 실제로 적용해 AI와 함께 '움직이는 시각화'를
직접 구현하는 법을 알아보겠습니다.
