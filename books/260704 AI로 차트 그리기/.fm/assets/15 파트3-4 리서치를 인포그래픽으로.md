지금까지 우리는 '이미 가지고 있는 텍스트나 데이터'를 활용해 차트와
슬라이드로 만드는 법을 살펴봤습니다. 이번에는 한 단계 더 나아가 인터넷의
최신 정보를 AI가 직접 검색해 그 내용을 가지고 '인포그래픽'을 만드는
방법을 알아보겠습니다. 즉, 단순 편집이 아니라 '리서치[(정보 찾기)]{.m} +
디자인[(시각화)]{.m}'을 한 번에 연결하는 가장 실용적인 업무 자동화
기술을 살펴볼 겁니다.

신제품을 홍보해야 하는데 '요즘 우리 타깃[(20대)]{.m}은 어디서 콘텐츠를
볼까? 유튜브? 인스타 릴스? 틱톡?' 이런 고민을 해본 적 있으시죠? 보통
이런 리서치는 몇 시간 동안 인터넷을 검색해 자료를 모으고, PPT를 열어
비교 장표를 만드는 '두 번의 작업'을 해야 합니다. 하지만 AI를 활용하면 이
두 과정을 단 5분으로 줄일 수 있습니다.

\

#### ['정보 리서치'와 '디자인 시각화'의 분리]{.u2} {#031.xhtml_sigil_toc_id_32}

\

우리는 AI에게 한 번에 모든 것을 시킬 수도 있지만, 더 나은 결과물을
위해서는 두 단계로 나누어 작업하는 것이 좋습니다.

**① 리서치 단계** : AI가 최신 정보를 검색하고 핵심만 요약하게 합니다.

**② 시각화 단계** : 리서치 결과를 '그대로' 가져와 인포그래픽으로
만듭니다.

\

이렇게 하면 AI가 찾은 리서치 내용이 부실하거나 내 의도와 다를 때
1단계에서 즉시 수정할 수 있습니다. 디자인까지 다 했는데 내용을 다시
바꾸는 비극을 막을 수 있죠.

\

#### [신제품 홍보 채널 '리서치' 및]{.u2} {#031.xhtml_sigil_toc_id_33 title="신제품 홍보 채널 ‘리서치’ 및 ‘인포그래픽’ 만들기"}

['인포그래픽' 만들기]{.u2}

\

그럼 여기에서는 우리 회사가 '20대 타깃 뷰티 신제품'을 출시한다는
가정하에, 효과적인 SNS 채널을 비교하는 인포그래픽을 만들어 보겠습니다.

\

**1단계. 딥 리서치로 채널 특징 요약하기**

먼저, 제미나이의 'Deep Research[(심층검색)]{.m}'를 활성화하고, ACTS
프레임워크로 명확하게 지시합니다.

\

::: tb1
[프롬프트 예시(ACTS)]{.cor1}

\

**A[(Actor - 역할)]{.m}** : 너는 최신 트렌드에 정통한 최고의 디지털
마케팅 전략가야.

**C[(Context - 맥락)]{.m}** : 우리는 '20대 타깃 뷰티 신제품' 론칭을
앞두고 있어. 경영진에게 보고할 SNS 마케팅 채널 비교 자료가 필요해. 어떤
채널에 집중해야할지 최종 의사결정에 사용할 핵심요약 자료야.

**T[(Task - 작업)]{.m}** : '유튜브, 인스타그램[(릴스)]{.m}, 틱톡' 3개
채널의 특징을 딥 리서치해서 각 채널별로 '① 주요 사용자층' '② 핵심 콘텐츠
형식' '③ 브랜딩/광고의 최대 강점'의 3가지 항목으로 비교 분석해 줘.

**S(Style - 스타일)** : 결과는 각 채널[(유튜브, 인스타그램,
틱톡)]{.m}별로 명확하게 구분해 줘. 각 항목[(사용자층, 콘텐츠 형식,
강점)]{.m}을 명확하게 식별할 수 있게 해주고, 문장은 경영진이 이해하기
쉽도록 간결하고 명확하게 요약해 줘.
:::

\

::: w100
![](AI로 차트 그리기/Images/186.jpg){.w100}
:::

\

'실행' 버튼을 누르면 제미나이가 검색을 시작하며, 연구 계획을 보여줍니다.
그럼 제미나이가 제안한 개요를 검토해, 혹시 수정할 부분이 있다면 수정을
하고, '연구 시작' 버튼을 누르면 됩니다.

참고로 '경영진 보고'처럼 단 하나의 실수도 용납되지 않는 중요한 리서치를
할 때에는 AI를 더 정직하게 만드는 '마법의 주문'을 추가하는 것이
좋습니다. 앞에서 배운 'AI 업무규칙' 기억하시죠. AI에게 중요한 리서치를
시킬 때마다 이 규칙을 프롬프트 마지막 S[(스타일]{.m}) 부분에 추가해
두세요.

\

::: tb1
[AI 업무규칙 프롬프트]{.cor1}

\

(출처 요구) "모든 정보와 데이터의 출처를 명확하게 제시해 줘."

(추측 방지) "추가 정보를 임의로 생성하지 말고, 주어진 데이터만 활용해
줘.

불확실한 정보가 있다면 '확인 필요'라고 반드시 표시해 줘."

(사실/추정 구분) "정보가 부족해 예측이 필요할 경우, '추정'이라고 명확히
표시해 줘. 임의대로 사실이라고 확정하면 안 돼."
:::

\

제미나이는 딥 리서치를 통해 3가지 SNS 채널의 특징을 요약했습니다.

**2단계. 리서치 결과를 인포그래픽으로 만들기**

보고서가 완성되고 내용에 이상이 없다면 오른쪽 상단의 '만들기' 버튼을
누릅니다. 바로 여기에 보고서 내용을 멋지게 만들어 줄 마법의 버튼이
있습니다. '인포그래픽' 버튼을 누르면 제미나이가 리서치 결과를 기반으로
코드를 짜서 보고서 내용을 깔끔한 인포그래픽으로 만들어 줍니다.

이때 인포그래픽이 마음에 들지 않으면 입력창에 "전체 제목 글씨 크기를
30px로 키워줘" "각 항목 사이의 간격을 좀 더 넓혀서 가독성을 높여줘"와
같이 수정사항을 입력하면 AI는 우리의 지시를 즉각 반영하여 코드를 수정해
줍니다.

#### [리서치와 디자인이]{.u2} {#031.xhtml_sigil_toc_id_34 title="리서치와 디자인이 하나의 흐름으로 연결되는 순간"}

[하나의 흐름으로 연결되는 순간]{.u2}

\

어떤가요? 우리는 단 몇 분 만에, 인터넷에서 최신 자료를 검색한 '리서치'
작업과, 그 결과를 PPT 장표로 만드는 '디자인' 작업을 동시에 끝냈습니다.

여기서 중요한 점은 단순히 예쁜 그림을 만든 것이 아니라, AI가 방금 찾아낸
최신 정보를 기반으로 보고서를 만들었다는 점입니다. 정보[(리서치)]{.m}와
형식[(디자인)]{.m}을 분리하여 생각하고, 이를 단계적으로 AI에게 지시하는
것. 이것이 바로 AI 시대의 센스 있는 업무방식입니다.
